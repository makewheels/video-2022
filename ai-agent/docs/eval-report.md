## 观察与分析

### 模型表现出色的方面

1. **工具选择准确**：38 个工具中，模型几乎每次都选对了最合适的
2. **写操作确认**：全部 13 个写操作场景都正确阻止并要求确认
3. **消歧自然**：遇到「AI 教程」这种模糊查询，列出两个候选让用户选
4. **数据准确**：回答中的数字 100% 来自工具结果，无虚构造假
5. **中文自然度**：回答格式清晰（表格、列表），带 emoji 但不过度

### 仍需改进

| 方向 | 说明 |
|------|------|
| **模型供应商切换** | 当前用 DeepSeek 测试，需切到 MiniMax 并重跑 eval |
| **多轮对话 E2E** | 当前 eval 只有单轮，无「追问→继承上下文」的场景 |
| **LLM Judge** | 当前 grader 是规则匹配，对措辞变化敏感。需要 LLM judge 评估回答质量 |
| **真实后端** | 49 条全跑在 fixture 上，需 seed 脚本对接真实测试环境 |
| **流式体验** | chat_stream 已实现但未在 eval 中验证 |
| **超时/重试** | 未测试网络异常、API 超时等边界 |

## 后续建议

1. **切 MiniMax 重跑**：拿到 API key 后跑同一套 eval，对比通过率
2. **加多轮场景**：10 条多轮对话 eval（消歧后追问、连续操作等）
3. **加真实后端**：写 seed 脚本 → `--backend cli` 自动跑
4. **加 LLM Judge**：评估「回答是否自然」「是否过度承诺」等代码难判的维度
5. **上线 Web Chat**：FastAPI 已有，前端接 chat UI

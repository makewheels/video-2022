# 用户模块测试

用户模块覆盖手机号验证码登录、会话管理和客户端标识三个核心功能。测试验证了从验证码发送到用户创建的完整登录流程，以及 Token 缓存、会话创建等关键行为。

- **测试文件数**：3
- **测试用例数**：23

## 运行方式

```bash
# 运行用户模块全部测试
export $(grep -v '^#' .env | grep -v '^$' | xargs) && mvn test -pl video -Pspringboot \
  -Dtest="com.github.makewheels.video2022.user.*" \
  -Dlog4j.configuration=file:///tmp/log4j.properties
```

## UserServiceTest

验证手机号验证码登录流程、用户创建/查询、Token 管理等核心用户功能。

| # | 测试方法 | 场景 | 前置条件 | 操作 | 期望结果 | 测试目标 |
|---|---------|------|---------|------|---------|---------|
| 1 | `requestVerificationCode_shouldStoreCodeInRedis` | 请求验证码 | 清空 Redis | 调用 `requestVerificationCode(phone)` | Redis 中存储 4 位验证码，手机号匹配 | 验证码生成与 Redis 存储 |
| 2 | `requestVerificationCode_calledTwice_shouldReturnSameCode` | 重复请求返回相同验证码 | 清空 Redis | 同一手机号调用两次 | 两次返回相同验证码 | 验证码缓存幂等性 |
| 3 | `submitVerificationCode_withValidCode_shouldCreateUserAndGenerateToken` | 正确验证码登录 | 已请求验证码 | 提交正确验证码 | 创建用户，生成 Token | 登录主流程 |
| 4 | `submitVerificationCode_withDevCode111_shouldSucceed` | 开发用测试码 111 | 已请求验证码 | 提交验证码 "111" | 登录成功 | 开发环境快捷登录 |
| 5 | `submitVerificationCode_withWrongCode_shouldThrowException` | 错误验证码 | 已请求验证码 | 提交错误码 "9999" | 抛出 `USER_PHONE_VERIFICATION_CODE_WRONG` | 错误验证码拒绝 |
| 6 | `submitVerificationCode_withExpiredCode_shouldThrowException` | 过期验证码 | 未请求验证码 | 直接提交验证码 | 抛出 `USER_PHONE_VERIFICATION_CODE_EXPIRED` | 过期验证码拒绝 |
| 7 | `submitVerificationCode_forExistingUser_shouldUpdateNotDuplicate` | 已有用户重新登录 | 用户已创建 | 同一手机号再次登录 | 用户 ID 不变，Token 刷新，数据库仅 1 条记录 | 避免重复创建用户 |
| 8 | `submitVerificationCode_shouldRemoveVerificationCodeFromRedis` | 登录后清除验证码 | 登录成功 | 检查 Redis | 验证码已从 Redis 删除 | 一次性验证码安全策略 |
| 9 | `getUserByToken_shouldReturnUserFromRedisCache` | Token 查询命中缓存 | 用户已登录，Token 在 Redis | 按 Token 查询用户 | 返回缓存中的用户，ID 和手机号正确 | Redis 缓存查询 |
| 10 | `getUserByToken_shouldFallBackToMongoDB` | Token 查询缓存未命中 | 用户在 MongoDB 中，Redis 已清空 | 按 Token 查询用户 | 从 MongoDB 查到用户 | 缓存穿透回退 |
| 11 | `getUserByToken_withInvalidToken_shouldReturnNull` | 无效 Token 查询 | Token 不存在 | 查询 "non-existent-token" | 返回 null | 无效 Token 处理 |
| 12 | `getUserByToken_withNullToken_shouldReturnNull` | Null Token 查询 | 无 | 传入 null | 返回 null | 空值安全处理 |
| 13 | `getUserById_shouldReturnUserWithoutToken` | 按 ID 查询用户脱敏 | 用户已创建 | 按用户 ID 查询 | 返回用户但 Token 为 null | Token 脱敏安全策略 |
| 14 | `getUserById_withNonExistentId_shouldReturnNull` | 不存在的用户 ID | 无 | 查询不存在的 ID | 返回 null | 缺失用户处理 |
| 15 | `getUserByToken_withEmptyStringToken_shouldReturnNull` | 空字符串 Token | 无 | 传入空字符串 | 返回 null | 空字符串安全处理 |
| 16 | `submitVerificationCode_withEmptyCode_shouldThrow` | 空验证码提交 | 已请求验证码 | 提交空字符串 | 抛出 `USER_PHONE_VERIFICATION_CODE_WRONG` | 空值验证 |
| 17 | `requestVerificationCode_differentPhones_shouldGenerateDifferentCodes` | 不同手机号生成不同验证码 | 清空状态 | 为两个手机号请求验证码 | 各自生成不同的验证码 | 手机号隔离 |

## SessionServiceTest

验证会话（Session）的创建和管理，每次请求生成独立的 sessionId。

| # | 测试方法 | 场景 | 前置条件 | 操作 | 期望结果 | 测试目标 |
|---|---------|------|---------|------|---------|---------|
| 1 | `requestSessionId_shouldCreateSessionAndReturnId` | 首次创建会话 | Mock HTTP 请求（IP: 192.168.1.100） | 调用 `requestSessionId()` | 返回非空 sessionId，MongoDB 中持久化 IP、User-Agent、createTime | 会话创建与持久化 |
| 2 | `requestSessionId_calledTwice_shouldCreateTwoDistinctSessions` | 多次请求生成不同会话 | Mock HTTP 请求 | 调用两次 `requestSessionId()` | 每次返回不同的 sessionId | 会话唯一性 |
| 3 | `requestSessionId_withNullUserAgent_shouldStillSucceed` | 缺少 User-Agent | Mock 请求无 User-Agent 头 | 调用 `requestSessionId()` | 返回有效 sessionId，User-Agent 为 null | 缺失 Header 容错 |

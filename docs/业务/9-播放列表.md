# 播放列表

> 文档地图：[README](../../README.md) > [关键设计](../1-关键设计.md) > 本文档

---

## 1. 功能概览

播放列表模块提供完整的 CRUD 能力，以及对列表内视频条目（PlayItem）的增 / 删 / 移动管理。

| 能力 | 说明 |
|------|------|
| 创建 / 更新 / 删除 / 恢复 播放列表 | 支持软删除与恢复 |
| 添加视频到播放列表 | 支持批量添加，可选插入到头部或尾部 |
| 删除播放列表中的视频 | 8 种删除模式（按 videoId / playItemId / 索引 / 范围 / 时间 / 全部） |
| 移动视频位置 | 5 种移动模式（指定索引 / 某视频前后 / 置顶 / 置底） |
| 查询播放列表 | 按 ID 查询、分页查询（自己的 / 指定用户的）、反向查询视频所属列表 |
| 获取列表条目播放详情 | 返回 PlayItemVO，携带视频标题、时长、封面、播放地址等 |

## 2. 接口列表

所有接口前缀：`/playlist`

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| POST | `/createPlaylist` | Body: `{title, description}` | 创建播放列表，title 必填 |
| POST | `/updatePlaylist` | Body: `{playlistId, title, description, visibility}` | 更新播放列表元信息，title 必填 |
| GET | `/deletePlaylist` | `playlistId` | 软删除播放列表（`deleted = true`） |
| GET | `/recoverPlaylist` | `playlistId` | 恢复已删除的播放列表（`deleted = false`） |
| GET | `/getPlaylistById` | `playlistId`, `showVideoList`（默认 false） | 按 ID 获取播放列表；`showVideoList=false` 时不返回 videoList |
| GET | `/getPlayItemListDetail` | `playlistId` | 获取列表内所有条目的播放详情（PlayItemVO） |
| GET | `/getPlaylistByPage` | `userId`, `skip`, `limit` | 分页获取指定用户的播放列表 |
| GET | `/getMyPlaylistByPage` | `skip`, `limit` | 分页获取当前登录用户的播放列表 |
| POST | `/addPlaylistItem` | Body: `{playlistId, videoIdList, addMode}` | 批量添加视频到播放列表 |
| POST | `/deletePlaylistItem` | Body: `{playlistId, deleteMode, videoIdList, playItemIdList, singleIndex, indexList, startIndex, endIndex, startTime, endTime}` | 按模式删除列表中的视频 |
| POST | `/movePlaylistItem` | Body: `{playlistId, videoId, moveMode, toIndex, toVideoId}` | 按模式移动视频位置 |
| GET | `/getPlaylistByVideoId` | `videoId` | 反向查询：该视频被当前用户加入了哪些播放列表 |

## 3. 排序逻辑

### 3.1 添加时的位置（AddMode）

| 模式 | 常量 | 行为 |
|------|------|------|
| 添加到头部 | `ADD_TO_TOP` | `idBeanList.add(0, idBean)` |
| 添加到尾部 | `ADD_TO_BOTTOM` | `idBeanList.add(idBean)` |

批量添加时逐条遍历 `videoIdList`，每条按 `addMode` 插入。

### 3.2 移动位置（MoveMode）

移动的核心实现在 `MovePlayItemService.movePlayItem()`：

1. 从 MongoDB 查出 Playlist 及其关联的所有 PlayItem 列表。
2. 根据 `videoId` 找到 **sourcePlayItem**。
3. 根据 `moveMode` 找到 **targetPlayItem**：

| 模式 | 常量 | 如何确定 targetPlayItem |
|------|------|------------------------|
| 移到指定索引 | `TO_INDEX` | `playItemList.get(toIndex)` |
| 移到某视频之前 | `BEFORE_VIDEO` | 找到 `videoId == toVideoId` 的 PlayItem |
| 移到某视频之后 | `AFTER_VIDEO` | 找到 `videoId == toVideoId` 的 PlayItem |
| 置顶 | `TO_TOP` | `playItemList.get(0)` |
| 置底 | `TO_BOTTOM` | `playItemList.get(size - 1)` |

4. 调用 `Collections.swap(playItemList, sourceIndex, targetIndex)` 交换两者位置。
5. 更新 Playlist 的 `updateTime` 并保存。

> **注意**：当前实现使用 `swap`（交换），而非"抽出再插入"。因此 `BEFORE_VIDEO` / `AFTER_VIDEO` 实际效果是 source 与 target **互换位置**，而非将 source 移到 target 前/后。

### 3.3 分页查询排序

`PlaylistService.getPlaylistByPage()` 在内存中按 `updateTime` 倒序排列返回结果。

## 4. 分享机制

播放列表通过 `visibility` 字段控制可见性，复用视频的 `Visibility` 常量：

| 值 | 含义 |
|----|------|
| `PUBLIC` | 完全公开 |
| `UNLISTED` | 有链接可访问，平台不可搜索 |
| `PRIVATE` | 仅拥有者登录后可见 |

- 创建时默认 `visibility = PUBLIC`。
- 更新时通过 `UpdatePlaylistRequest.visibility` 修改。
- `getPlaylistByPage` 接口接受任意 `userId` 参数，因此可以获取指定用户的播放列表（分页查询条件：`ownerId == userId && isDelete == false`）。
- `getMyPlaylistByPage` 是语法糖，自动取 `UserHolder.getUserId()` 作为 userId。
- `getPlaylistByVideoId` 反向查询当前用户拥有的 PlayItem 中包含指定 videoId 的记录，返回去重后的 playlistId 列表。

## 5. 数据模型

### 5.1 Playlist（MongoDB Document）

| 字段 | 类型 | 索引 | 说明 |
|------|------|------|------|
| `id` | String | 主键 | MongoDB `_id` |
| `title` | String | — | 播放列表标题 |
| `description` | String | — | 描述 |
| `ownerId` | String | ✅ | 所属用户 ID |
| `visibility` | String | ✅ | PUBLIC / UNLISTED / PRIVATE |
| `deleted` | Boolean | ✅ | 软删除标记，默认 `false` |
| `createTime` | Date | ✅ | 创建时间，构造时初始化 |
| `updateTime` | Date | ✅ | 更新时间，构造时初始化 |
| `videoList` | `List<IdBean>` | — | 有序的视频引用列表 |

### 5.2 IdBean（内嵌对象）

| 字段 | 类型 | 说明 |
|------|------|------|
| `playItemId` | String | 对应 PlayItem 的 ID |
| `videoId` | String | 对应 Video 的 ID |

`Playlist.videoList` 维护了播放列表的**顺序**，是排序的唯一来源。

### 5.3 PlayItem（MongoDB Document）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 主键 |
| `playlistId` | String | 所属播放列表 ID |
| `videoId` | String | 关联视频 ID |
| `owner` | String | 创建者用户 ID |
| `isDelete` | Boolean | 软删除标记，默认 `false` |
| `createTime` | Date | 创建时间 |
| `updateTime` | Date | 更新时间 |

### 5.4 PlayItemVO（视图对象）

用于 `getPlayItemListDetail` 接口返回，聚合了 PlayItem 与 Video 信息：

| 字段 | 来源 |
|------|------|
| `playItemId` | IdBean.playItemId |
| `videoId` | IdBean.videoId |
| `title`, `duration`, `coverUrl`, `watchCount` | Video 实体（BeanUtils.copyProperties） |
| `watchId`, `watchUrl`, `shortUrl` | Video 实体 |
| `videoStatus` | Video.status |
| `videoCreateTime`, `videoUpdateTime` | Video 时间格式化 |

### 5.5 软删除

- **Playlist**：`deleted` 字段。删除时设为 `true`，恢复时设为 `false`。分页查询过滤 `isDelete == false`。
- **PlayItem**：`isDelete` 字段。删除条目时设为 `true`，并从 Playlist 的 `videoList` 中移除对应 IdBean。

## 6. 代码调用链

### 6.1 创建播放列表

```
PlaylistController.createPlaylist()
  → PlaylistService.createPlaylist()
    → new Playlist() 设置 title/description/ownerId/visibility
    → PlaylistRepository.save(playlist)          // MongoTemplate.save
```

### 6.2 更新播放列表

```
PlaylistController.updatePlaylist()
  → PlaylistService.updatePlaylist()
    → CheckService.checkPlaylistOwner()          // 权限校验
    → CheckService.checkVideoVisibility()        // visibility 合法性校验
    → PlaylistRepository.getPlaylist()
    → 更新字段 → PlaylistRepository.save()
```

### 6.3 删除 / 恢复播放列表

```
PlaylistController.deletePlaylist()
  → PlaylistService.deletePlaylist()
    → CheckService.checkPlaylistOwner()
    → playlist.setDeleted(true) → PlaylistRepository.save()

PlaylistController.recoverPlaylist()
  → PlaylistService.recoverPlaylist()
    → CheckService.checkPlaylistCanDelete()
    → playlist.setDeleted(false) → PlaylistRepository.save()
```

### 6.4 添加视频到播放列表

```
PlaylistController.addPlaylistItem()
  → PlayItemService.addVideoToPlaylist()
    → CheckService.checkPlaylistOwner()
    → CheckService.checkVideoNotBelongToPlaylist()   // 防重复
    → CheckService.checkPlaylistAddMode()
    → AddPlayItemService.addVideoToPlaylist()
      → 遍历 videoIdList:
        → new PlayItem() → PlaylistRepository.save(playItem)
        → PlaylistRepository.getPlaylist()
        → 按 AddMode 插入 IdBean 到 videoList
        → PlaylistRepository.save(playlist)
```

### 6.5 删除列表中的视频

```
PlaylistController.deletePlaylistItem()
  → PlayItemService.deletePlayItem()
    → CheckService.checkPlaylistOwner()
    → CheckService.checkVideoBelongToPlaylist()
    → CheckService.checkDeletePlayItemMode()
    → DeletePlayItemService.deletePlayItem()
      → getDeletePlayItems()                     // 按 DeleteMode 筛选目标
      → 遍历: playItem.setIsDelete(true) → save
      → playlist.videoList.removeIf(匹配的 IdBean)
    → PlaylistRepository.save(playlist)
```

### 6.6 移动视频位置

```
PlaylistController.movePlaylistItem()
  → PlayItemService.movePlayItem()
    → CheckService.checkPlaylistOwner()
    → CheckService.checkVideoBelongToPlaylist()
    → CheckService.checkPlaylistItemMoveMode()
    → MovePlayItemService.movePlayItem()
      → PlaylistRepository.getPlaylist()
      → PlaylistRepository.getPlayItemList()     // 按 videoList 顺序排序
      → 找 sourcePlayItem（按 videoId）
      → findTargetPlayItem()（按 MoveMode）
      → Collections.swap(source, target)
      → PlaylistRepository.save(playlist)
```

### 6.7 获取播放详情

```
PlaylistController.getPlayItemListDetail()
  → PlayItemService.getPlayItemListDetail()
    → CheckService.checkPlaylistExist()
    → PlaylistRepository.getPlaylist()
    → 提取 videoIdList → VideoRepository.getMapByIdList()
    → 遍历 videoList，组装 PlayItemVO（BeanUtils.copyProperties）
```

### 6.8 反向查询视频所属列表

```
PlaylistController.getPlaylistByVideoId()
  → PlaylistService.getPlaylistByVideoId()
    → PlaylistRepository.getPlaylistByVideoAndUser()
      → 查 PlayItem: videoId == X && owner == userId
      → 返回去重的 playlistId 列表
```

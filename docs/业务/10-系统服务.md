## 1. ID 生成策略

> 源码：`utils/IdService.java`、`etc/redis/RedisKey.java`

### 1.1 核心机制

系统所有业务 ID 均通过 `IdService`（`@Service`，所有方法 `synchronized`）生成，依赖 **Redis 原子自增计数器** 保证全局唯一性。

### 1.2 Long ID 生成算法

方法 `nextId(Duration duration, int serialNumberLength, int randomLength)`：

```
ID = timeUnit + serialNumber + random
```

| 分段 | 说明 |
|------|------|
| `timeUnit` | `(当前毫秒时间戳 - 起始时间) / duration毫秒`。起始时间为 `2023-01-01 00:00:00 UTC`。 |
| `serialNumber` | Redis 原子自增序列号。Key 为 `video-2022:increaseLongId:{timeUnit}`，TTL 为 `duration × 2`。用 `DecimalFormat` 补零到 `serialNumberLength` 位。 |
| `random` | `RandomUtil.randomNumbers(randomLength)` 生成的纯数字随机尾缀。 |

**默认 Long ID**（`nextLongId()`）参数：`duration=4s`、`serialNumberLength=5`、`randomLength=3`。

即：每 4 秒一个时间窗口，窗口内最多 99999 个序列号，加 3 位随机数。

### 1.3 Short ID 生成算法（watchId）

方法 `nextShortId()`：

1. **时间戳**：`(当前UTC秒 - 1640995200) / 86400`，即以 2022-01-01 零点为起始，按天递增。
2. **序列号**：Redis Key `video-2022:increaseShortId:{yyyy-MM-dd}`，自增后 `DecimalFormat("00")` 补零，TTL 2 天。
3. **随机尾缀**：3 位随机数字。
4. **拼接**：`timestamp + increaseId + random` 转为十进制 `long`，再 `Long.toString(decimal, Character.MAX_RADIX).toUpperCase()` 转为 **36 进制大写字符串**。

结果是一个紧凑的短字符串，用作视频的 `watchId`。

### 1.4 所有 ID 类型

| 方法 | 前缀 | 用途 | 示例 |
|------|------|------|------|
| `getUserId()` | `u_` | 用户 ID | `u_18432000540012345678` |
| `getVideoId()` | `v_` | 视频 ID | `v_18432000540012345678` |
| `getFileId()` | `f_` | 文件 ID | `f_18432000540012345678` |
| `getTsFileId()` | `f_ts_` | TS 切片文件 ID | `f_ts_18432000540012345678` |
| `getTranscodeId()` | `tr_` | 转码任务 ID | `tr_18432000540012345678` |
| `getCoverId()` | `c_` | 封面 ID | `c_18432000540012345678` |
| `nextLongId(prefix)` | 自定义 | 通用带前缀 ID | `{prefix}18432000540012345678` |
| `nextShortId()` | 无 | watchId（36 进制短码） | `1A2B3C4D` |

> 所有带前缀的 ID 内部调用 `getCommonId()` → `nextLongId()`。

## 2. 异常处理

> 源码：`springboot/exception/` 包

### 2.1 GlobalExceptionHandler

`@ControllerAdvice` 全局异常拦截器，处理两类异常：

| 异常类型 | Handler | 返回 |
|----------|---------|------|
| `Exception.class` | `exceptionHandler(Exception)` | `Result(ErrorCode.FAIL.code, message, stackTrace)` |
| `VideoException.class` | `exceptionHandler(VideoException)` | `Result(errorCode.code, errorCode.message, stackTrace)` |

### 2.2 异常处理流程

```
异常发生
  → 打印堆栈
  → handleException()
      → 构造 ExceptionLog 对象（含 systemErrorCode、systemErrorMessage、exceptionMessage、exceptionStackTrace、createTime）
      → mongoTemplate.save(exceptionLog) 持久化到 MongoDB
  → 返回 Result（含 code、message、stackTrace）
```

### 2.3 VideoException

自定义业务异常，持有 `ErrorCode` 枚举。支持三种构造：
- `VideoException(ErrorCode errorCode)`
- `VideoException(String message)` — 默认 `ErrorCode.FAIL`
- `VideoException(ErrorCode errorCode, String message)`

### 2.4 ExceptionLog 实体

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | MongoDB 自动生成 |
| `systemErrorCode` | Integer | 业务错误码（仅 VideoException） |
| `systemErrorMessage` | String | 业务错误消息（仅 VideoException） |
| `exceptionMessage` | String | `exception.getMessage()` |
| `exceptionStackTrace` | String | 完整堆栈字符串 |
| `createTime` | Date | 构造时自动设置 `new Date()` |

### 2.5 ExceptionController

`GET /exceptionLog/getById?exceptionLogId={id}` — 按 ID 查询异常日志。

### 2.6 ErrorCode 枚举

| 分组 | code | 含义 |
|------|------|------|
| 通用 | 0 | 成功 |
| 通用 | 1 | 未知错误 |
| 用户 | 11-15 | 验证码错误/过期、token 校验失败、用户不存在/未登录 |
| 视频 | 21-25 | 创建参数不合法、视频不存在、上传者不匹配、未就绪/已就绪 |
| 文件 | 31-34 | 文件不存在/未就绪/用户不匹配/上传凭证失败 |
| 封面 | 41 | 封面不存在 |
| 播放列表 | 51-53 | 不存在/已删除/用户不匹配 |

## 3. IP 地理位置

> 源码：`utils/IpService.java`

### 3.1 流程

```
getIpWithRedis(ip)
  → 查 Redis 缓存 key: video-2022:ip:{ip}
  → 若命中 → 直接返回 JSONObject
  → 若未命中
      → 调用阿里云 API 网关 https://ips.market.alicloudapi.com/iplocaltion?ip={ip}
         Header: Authorization: APPCODE {appcode}
      → handleApiResponse(): 将嵌套的 result 对象展平到顶层，添加 ip 字段
      → 写入 Redis，TTL = 6 小时（RedisTime.SIX_HOURS = 21600 秒）
      → 返回 JSONObject
```

### 3.2 返回字段

```json
{
  "code": 100,
  "message": "success",
  "ip": "45.78.33.111",
  "en_short": "CN",
  "en_name": "China",
  "nation": "中国",
  "province": "香港特别行政区",
  "city": "",
  "district": "",
  "adcode": "810000",
  "lat": 22.27628,
  "lng": 114.16383
}
```

## 7. Redis Key 设计

> 源码：`etc/redis/RedisKey.java`、`etc/redis/RedisTime.java`

### 7.1 Key 总表

| Key 模式 | 方法 | TTL | 用途 | 使用位置 |
|----------|------|-----|------|----------|
| `video-2022:ip:{ip}` | `RedisKey.ip(ip)` | 6 小时 | IP 地理位置信息缓存。IPv6 冒号替换为下划线 | `IpService.getIpWithRedis()` |
| `video-2022:user:verificationCode:{phone}` | `RedisKey.verificationCode(phone)` | 10 分钟 | 手机验证码（JSON 序列化的 `VerificationCode` 对象） | `UserRedisService` |
| `video-2022:user:token:{token}` | `RedisKey.token(token)` | 30 分钟 | 用户登录 Token → User 对象 JSON | `UserRedisService` |
| `video-2022:increaseShortId:{yyyy-MM-dd}` | `RedisKey.increaseShortId()` | 2 天 | Short ID（watchId）每日自增序列号 | `IdService.nextShortId()` |
| `video-2022:increaseLongId:{timeUnit}` | `RedisKey.increaseLongId(timeUnit)` | 8 秒（duration × 2） | Long ID 每时间窗口自增序列号 | `IdService.nextLongId()` |

### 7.2 RedisTime 常量（单位：秒）

| 常量 | 值 | 秒数 |
|------|------|------|
| `ONE_MINUTE` | `60` | 60 |
| `TEN_MINUTES` | `10 × 60` | 600 |
| `THIRTY_MINUTES` | `30 × 60` | 1,800 |
| `ONE_HOUR` | `60 × 60` | 3,600 |
| `TWO_HOURS` | `2 × 3,600` | 7,200 |
| `THREE_HOURS` | `3 × 3,600` | 10,800 |
| `SIX_HOURS` | `6 × 3,600` | 21,600 |
| `ONE_DAY` | `24 × 3,600` | 86,400 |
| `TWO_DAYS` | `2 × 86,400` | 172,800 |

### 7.3 Key 前缀规范

所有 Key 以 `video-2022` 为根前缀，使用冒号 `:` 分隔层级：

```
video-2022                       # 根
video-2022:user                  # 用户相关
video-2022:user:verificationCode # 验证码
video-2022:user:token            # 登录 token
video-2022:ip                    # IP 缓存
video-2022:increaseShortId       # Short ID 计数器
video-2022:increaseLongId        # Long ID 计数器
```
